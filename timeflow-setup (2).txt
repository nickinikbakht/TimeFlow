## 🔥 Firebase Setup (Required for Cloud Sync)

### Step 1: Create Firebase Project

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Click **"Add project"** or **"Create a project"**
3. Enter project name: `TimeFlow` (or any name you prefer)
4. **Disable Google Analytics** (optional, not needed for this app)
5. Click **"Create project"**
6. Wait for setup to complete, then click **"Continue"**

### Step 2: Enable Authentication

1. In Firebase Console, click **"Authentication"** in the left sidebar
2. Click **"Get started"**
3. Click **"Sign-in method"** tab
4. Click on **"Email/Password"**
5. Toggle **"Enable"** switch ON
6. Click **"Save"**

### Step 3: Enable Realtime Database

1. Click **"Realtime Database"** in the left sidebar
2. Click **"Create Database"**
3. Select your location (choose closest to you)
4. Start in **"Test mode"** (we'll secure it later)
5. Click **"Enable"**

### Step 4: Configure Database Security Rules

1. In Realtime Database, click **"Rules"** tab
2. Replace the rules with:

```json
{
  "rules": {
    "users": {
      "$uid": {
        ".read": "$uid === auth.uid",
        ".write": "$uid === auth.uid"
      }
    }
  }
}
```

3. Click **"Publish"**

This ensures users can only read/write their own data.

### Step 5: Get Your Configuration

1. Click the **gear icon ⚙️** (top left) → **"Project settings"**
2. Scroll down to **"Your apps"** section
3. Click the **web icon** `</>`
4. Register app nickname: `TimeFlow`
5. **DO NOT** check "Firebase Hosting"
6. Click **"Register app"**
7. Copy the `firebaseConfig` object

### Step 6: Update app.js

1. Open `app.js` in a text editor
2. Find lines 8-15 (the firebaseConfig section)
3. Replace with YOUR config from Firebase:

```javascript
const firebaseConfig = {
    apiKey: "YOUR_API_KEY_HERE",
    authDomain: "your-project.firebaseapp.com",
    databaseURL: "https://your-project-default-rtdb.firebaseio.com",
    projectId: "your-project",
    storageBucket: "your-project.appspot.com",
    messagingSenderId: "123456789012",
    appId: "1:123456789012:web:abcdef123456"
};
```

4. Save the file

## 🚀 Running the App

### Option 1: Local Development (Simple)

1. Open `index.html` in your web browser
2. That's it! The app will work locally

### Option 2: Local Server (Recommended)

If you have Python installed:

```bash
# Python 3
python -m http.server 8000

# Then open: http://localhost:8000
```

If you have Node.js installed:

```bash
# Install http-server globally
npm install -g http-server

# Run server
http-server

# Then open: http://localhost:8080
```

### Option 3: Deploy Online (Free)

**Using Firebase Hosting:**

```bash
# Install Firebase CLI
npm install -g firebase-tools

# Login to Firebase
firebase login

# Initialize Firebase in your project folder
firebase init hosting

# Select your project
# Set public directory to: . (current directory)
# Configure as single-page app: No

# Deploy
firebase deploy
```

**Using Netlify (Drag & Drop):**

1. Go to [Netlify](https://www.netlify.com/)
2. Drag and drop your `timeflow` folder
3. Done! You'll get a URL like: `your-app.netlify.app`

**Using GitHub Pages:**

1. Create a GitHub repository
2. Upload all files
3. Go to Settings → Pages
4. Select branch: `main`, folder: `/root`
5. Save and get your URL

## 🔐 User Authentication

### Creating an Account

1. Open the app
2. Click **"Sign Up"** tab
3. Enter:
   - Full Name
   - Email
   - Password (minimum 6 characters)
   - Confirm Password
4. Click **"Create Account"**
5. You'll be automatically logged in

### Signing In

1. Open the app
2. Click **"Sign In"** tab (default)
3. Enter email and password
4. Click **"Sign In"**

### Password Reset

Firebase handles password reset automatically. To enable:

1. Go to Firebase Console → Authentication
2. Click "Templates" tab
3. Customize the password reset email

To trigger password reset:
- Add this button to your login page if needed
- Or users can use Firebase's built-in recovery

## 📱 Features Overview

### 1. Tasks Tab
- Create tasks with priorities (High/Medium/Low)
- Add categories
- Filter tasks (All, Active, Completed, High Priority)
- Mark tasks as complete
- Delete tasks

### 2. Schedule Tab
- Schedule events with date and time
- Set recurring events (Daily, Weekly, Monthly)
- Categorize events
- View upcoming events sorted by date

### 3. Journal Tab
- Record activities with start and end times
- Choose from past activities (quick select)
- Add notes and tags
- View activities grouped by day
- Automatically sorted by start time
- **Overlap detection** - warns when activities overlap

### 4. Timeline Tab
- Visual timeline view of activities for any day
- Shows time blocks for each activity
- **Highlights overlapping activities in red**
- Select any date to view its timeline

### 5. Today Tab
- Quick overview of today's schedule
- Today's events
- Pending tasks
- Today's activities with total time tracked

### 6. Analytics Tab
- Task completion rate
- Task breakdown by priority
- Total activities logged
- **Time tracking by activity** with visual charts
- Total time tracked across all activities

## 🔒 Hidden Settings (Delete All Data)

To access the hidden "Danger Zone" settings:

1. Go to **Analytics** tab
2. Press: **Shift + Ctrl + R** three times quickly
3. A red "Danger Zone" card will appear
4. Click "Delete All My Data"
5. Type "DELETE" to confirm
6. All your data will be permanently deleted

**Note:** This only deletes YOUR data, not other users' data.

## ⚠️ Overlap Detection

The app automatically detects when activities overlap:

- **Journal Tab**: Shows overlap warning badges
- **Timeline Tab**: Highlights overlapping activities in red
- Visual warnings when activities conflict

This helps you identify scheduling mistakes or multitasking.

## 📊 Data Structure

Your data is stored in Firebase under:

```
/users
  /{userId}
    /profile
      - name
      - email
      - createdAt
    /data
      /tasks
        - id, text, completed, priority, category, createdAt
      /events
        - id, title, date, time, recurring, category, createdAt
      /journals
        - id, title, content, tags, date, startTime, endTime, duration, createdAt
```

## 🔧 Troubleshooting

### "Firebase not configured" error
- Check that you've replaced the firebaseConfig in `app.js`
- Ensure all values are correct (no "YOUR_API_KEY_HERE" placeholders)

### Can't sign up / sign in
- Check that Email/Password authentication is enabled in Firebase Console
- Check browser console for specific error messages

### Data not saving
- Check Firebase Realtime Database rules are set correctly
- Ensure you're signed in
- Check browser console for errors

### Overlap detection not working
- Ensure activities have the same date
- Check that start and end times are valid

## 📱 Mobile PWA Setup (Optional)

To make it installable on phones, create `manifest.json`:

```json
{
  "name": "TimeFlow",
  "short_name": "TimeFlow",
  "description": "Smart Schedule & Time Management",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#667eea",
  "theme_color": "#667eea",
  "icons": [
    {
      "src": "icon-192.png",
      "sizes": "192x192",
      "type": "image/png"
    },
    {
      "src": "icon-512.png",
      "sizes": "512x512",
      "type": "image/png"
    }
  ]
}
```

Add to `index.html` `<head>`:
```html
<link rel="manifest" href="manifest.json">
<meta name="theme-color" content="#667eea">
```
